# 네이버 부동산 크롤러 프로젝트 가이드

이 문서는 네이버 부동산 크롤러 프로젝트의 사용법, 주요 기능, 그리고 유지보수 방법을 설명합니다.

## 1. 프로젝트 개요

이 프로젝트는 네이버 부동산의 아파트 단지 정보를 크롤링하여 로컬 데이터베이스에 저장하고, 시계열 가격 변동 추이와 매물 현황을 시각화하여 보여주는 도구입니다.

### 주요 기능
- **단지 관리**: 관심 아파트 단지 등록, 수정, 삭제
- **데이터 수집**: 단지 기본 정보 및 실시간 매물 정보 크롤링
- **데이터 시각화**: 날짜별 매물 가격(최저/최고/평균) 및 물량 추이 그래프
- **필터링**: 매매/전세/월세, 면적별, 지하철 노선별 필터링

---

## 2. 시작하기 (Getting Started)

### 필수 요구 사항
- Node.js (v18 이상 권장)
- npm

### 설치 및 실행

1. **의존성 설치**
   프로젝트 루트에서 다음 명령어를 실행하여 전체 의존성을 설치합니다.
   ```bash
   npm install
   cd frontend && npm install
   cd ../backend && npm install
   cd ..
   ```

2. **데이터베이스 설정**
   SQLite 데이터베이스를 초기화합니다.
   ```bash
   cd backend
   npx prisma migrate dev
   cd ..
   ```

3. **프로젝트 실행**
   루트 디렉토리에서 다음 명령어로 프론트엔드와 백엔드를 동시에 실행합니다.
   ```bash
   npm run dev
   ```
   - Frontend: http://localhost:5173
   - Backend: http://localhost:3000

## 3. 사용 가이드

### 3.1 단지 등록하기
1. 메인 화면 우측 상단의 **'단지 추가'** 버튼을 클릭합니다.
2. **단지명**과 **네이버 단지 ID**를 입력합니다.
   - *네이버 단지 ID 찾는 법*: 네이버 부동산에서 해당 아파트를 검색한 후, URL의 `complex/` 뒤에 있는 숫자를 복사합니다.
   - 예: `https://new.land.naver.com/complexes/12345` -> ID는 `12345`

### 3.2 데이터 수집
상세 페이지에서 두 가지 수집 기능을 사용할 수 있습니다.
- **단지 정보 수집**: 세대수, 준공년도, 용적률 등 기본 정보를 가져옵니다. (최초 1회 권장)
- **매물 크롤링**: 현재 올라와 있는 매물 정보를 수집합니다. 주기적으로 실행하여 시세 데이터를 축적할 수 있습니다.

### 3.3 데이터 분석 및 시각화
- **차트**: 상세 페이지 상단의 그래프를 통해 가격 흐름을 확인합니다.
  - 기간 선택: 7일, 30일, 90일, 180일, 1년
  - 필터: 매매/전세/월세 및 평형 선택
- **매물 목록**: 하단 테이블에서 개별 매물 정보를 확인하고 정렬할 수 있습니다.

## 4. 유지보수 가이드 (Maintenance)

### 4.1 지하철 노선 목록 수정
새로운 지하철 노선(예: GTX-B, C 등)이 개통되어 목록에 추가해야 할 경우, 다음 두 파일을 수정해야 합니다.

1. **파일 위치**:
   - `frontend/src/pages/ComplexList.tsx`
   - `frontend/src/pages/ComplexDetail.tsx`

2. **수정 방법**:
   `SUBWAY_LINES` 배열에 새로운 노선명을 추가합니다.
   ```typescript
   const SUBWAY_LINES = [
     '1호선', ..., 'GTX-A', '새로운노선' // 여기에 추가
   ]
   ```
   
3. **색상 지정 (선택사항)**:
   `getTagColor` 함수에서 해당 노선의 뱃지 색상을 지정할 수 있습니다.

### 4.2 데이터베이스 스키마 변경
새로운 필드(예: 주차대수, 현관구조 등)를 저장하고 싶다면 Prisma 스키마를 수정해야 합니다.

1. **스키마 파일**: `backend/prisma/schema.prisma`
2. **모델 수정**: `Complex` 또는 `Listing` 모델에 필드 추가
   ```prisma
   model Complex {
     // ... 기존 필드
     parkingCount Int? // 예: 주차대수 추가
   }
   ```
3. **마이그레이션 실행**:
   ```bash
   cd backend
   npx prisma migrate dev --name add_parking_count
   ```
4. **백엔드/프론트엔드 코드 반영**: API 및 UI 코드 업데이트

### 4.3 크롤러 로직 수정
네이버 부동산 웹사이트의 구조(DOM)가 변경되어 크롤링이 실패할 경우, 크롤러 로직을 수정해야 합니다.

- **파일 위치**: `backend/src/scrapers/naverScraper.ts`
- **주요 포인트**: `page.evaluate` 내부의 `querySelector` 선택자가 네이버의 실제 HTML 클래스명과 일치하는지 확인하세요.

### 4.4 빌드 및 배포
프로젝트를 서버에 배포할 때는 프로덕션 빌드를 생성합니다.

```bash
# 전체 빌드
npm run build

# 실행 (백엔드)
cd backend
npm start
```
프론트엔드 빌드 결과물(`frontend/dist`)은 웹 서버(Nginx 등)를 통해 서빙하거나, 백엔드 Express 서버에서 정적 파일로 서빙하도록 설정할 수 있습니다.

## 5. 트러블슈팅

### Q. "Database is locked" 에러가 발생해요.
SQLite는 동시에 여러 쓰기 작업을 처리할 때 잠길 수 있습니다.
- **해결**: 크롤링이 진행 중일 때는 다른 수정 작업을 잠시 멈춰주세요. 문제가 지속되면 백엔드 서버를 재시작하세요.

### Q. 크롤링이 작동하지 않아요.
- **원인 1**: 네이버 단지 ID가 잘못되었을 수 있습니다. 링크를 확인해 보세요.
- **원인 2**: 네이버 부동산 페이지 구조가 변경되었을 수 있습니다. `naverScraper.ts`를 점검해야 합니다.
- **원인 3**: 너무 잦은 요청으로 IP가 차단되었을 수 있습니다. 잠시 후 다시 시도하세요.

### Q. 그래프에 데이터가 안 나와요.
- 매물 크롤링을 최소 2회 이상(다른 날짜에) 수행해야 의미 있는 변화 그래프가 그려집니다.
- 필터(매매/전세/월세)가 데이터가 있는 유형으로 선택되었는지 확인하세요.
